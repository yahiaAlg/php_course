SELECT post_id, title, created_at,
    TIMESTAMPDIFF(HOUR, created_at, NOW()) AS hours_ago,
    CASE
      WHEN TIMESTAMPDIFF(HOUR, created_at, NOW()) < 1 THEN 'Just now'
      WHEN TIMESTAMPDIFF(HOUR, created_at, NOW()) < 24 THEN 'Recent'
      ELSE 'Older'
    END AS status
FROM blog_posts
ORDER BY created_at DESC;

üéØ How TIMESTAMPDIFF Works:

TIMESTAMPDIFF(unit, start_date, end_date) calculates the difference between two timestamps in the specified unit.

HOUR
TIMESTAMPDIFF(HOUR, created_at, NOW())
MINUTE
TIMESTAMPDIFF(MINUTE, created_at, NOW())
DAY
TIMESTAMPDIFF(DAY, created_at, NOW())
MONTH
TIMESTAMPDIFF(MONTH, created_at, NOW())
YEAR
TIMESTAMPDIFF(YEAR, created_at, NOW())
SECOND
TIMESTAMPDIFF(SECOND, created_at, NOW())

üí° Common Use Cases:

  • "Time Ago" Display: Show "Posted 3 hours ago" in social media feeds
  • Session Timeouts: Find users inactive for more than 30 minutes
  • Data Freshness: Identify stale data that needs updating
  • SLA Monitoring: Track response times and missed deadlines
  • Age Calculations: Calculate how old records or users are

üîß Key Points:

  • Order Matters: TIMESTAMPDIFF(unit, older_date, newer_date) gives positive values
  • NOW() is Dynamic: Result changes each time the query runs
  • Truncated Results: Returns whole numbers (8.7 hours becomes 8)
  • Negative Values: If start_date > end_date, result is negative
