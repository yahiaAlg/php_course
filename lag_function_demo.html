Understanding LAG(price, 1) OVER (ORDER BY created_at)

SELECT product_id, created_at, price, LAG(price, 1) OVER (ORDER BY created_at) AS previous_price FROM products ORDER BY created_at;

ðŸŽ¯ How LAG Function Works:

LAG(price, 1) looks at the previous row (1 row back) in the ordered dataset and retrieves the price value.

  • First Row: Returns NULL because there's no previous row
  • Second Row: Returns the price from the first row (29.99)
  • Third Row: Returns the price from the second row (45.50)
  • And so on... Each row gets the price from the chronologically previous entry

ðŸ’¡ Common Use Cases:

  • Price Change Analysis: Compare current price vs previous price
  • Trend Detection: Identify increasing/decreasing patterns
  • Performance Metrics: Calculate period-over-period changes
  • Time Series Analysis: Analyze sequential data points

ðŸ”§ Syntax Breakdown:

LAG(column, offset) OVER (ORDER BY sort_column)

  • column: The column whose previous value you want
  • offset: How many rows back to look (1 = previous row)
  • ORDER BY: Defines the sequence for "previous"
